> **版本**：v1.0 | **状态**：MVP 开发中 | **作者**：AI PM 求职作品集 | **日期**：2026-06-06

## 1. 项目背景与问题定义

### 1.1 为什么要做这个项目？

产品经理最核心的工作之一是**理解用户**。用户访谈是最常用的定性研究方法，但存在一个尖锐的效率鸿沟：

> **1 小时访谈录音 → 整理成可用洞察需要 3-5 小时。10 个用户访谈下来，一周就没了。**

更致命的是，大多数 PM 不做结构化分析——他们凭"印象"回忆几个关键句，导致：
- 遗漏关键信号（某个用户提到的问题被忘记）
- 确认偏误（只记住符合自己预设的观点）
- 无法量化优先级（"好多用户都这么说"其实是几个人？）
- 团队协作困难（你的笔记别人看不懂）

### 1.2 现有解决方案的不足

| 用户现状 | 问题 |
|---------|------|
| **手动记笔记 + 回听录音** | 效率极低，1 小时访谈需要 3-5 小时整理 |
| **飞书妙记 / 通义听悟** | 只做语音转写，不做分析——"把录音变成文字，但文字还是得自己看" |
| **Dovetail / Condense** | 功能太重、英文优先、价格贵（$30+/月），不适合中国 PM 的工作节奏 |
| **ChatGPT 通用聊天** | 不结构化、每次都要重写 prompt、无法管理多个访谈项目 |
| **Excel + 手动编码** | 学术方法，不适合 PM 的快速迭代节奏 |

### 1.3 核心洞察

> **转写只是第一步。从"文字"到"洞察"的跨越，才是 PM 真正的价值所在——也是 AI 最应该代劳的部分。**

市场需要一个：面向中国 PM 的、轻量级的、AI 原生的、从转写到分析一站式完成的用户洞察工具。

## 2. 用户分析

### 2.1 目标用户画像

| 维度 | 描述 |
|------|------|
| **核心用户** | 1-5 年经验的互联网产品经理 |
| **使用场景** | 用户访谈后需要快速产出分析报告 |
| **频率** | 每月 2-8 次用户访谈 |
| **痛点** | "我知道用户说了什么，但我没时间系统分析" |
| **当前方案** | 飞书妙记转写 + 手动总结 + 印象回忆 |
| **技术熟练度** | 会用 SaaS 工具，不写代码 |
| **付费意愿** | 个人免费、团队付费 |

### 2.2 用户旅程

```
用户刚做完一场用户访谈（45 分钟录音）
    ↓
打开 InsightFlow
    ↓
上传录音文件 / 粘贴访谈文本
    ↓
给项目起个名字："外卖用户支付环节访谈"
    ↓
点击「开始分析」
    ↓
等待 1-3 分钟（AI 分析中，展示进度动画）
    ↓
收到一份结构化的 PM 洞察报告
    ↓
查看各维度分析结果
    ↓
导出/分享给团队成员
```

## 3. 产品目标与成功指标

### 3.1 产品目标

| 目标 | 描述 |
|------|------|
| **10x 效率提升** | 从"3 小时整理"到"5 分钟出报告" |
| **标准化输出** | 每次分析按统一框架产出，团队协作友好 |
| **PM 语言** | 输出 PM 看得懂、用得上的内容，不是学术论文 |
| **零学习成本** | 上传即用，不需要学任何概念 |

### 3.2 MVP 成功指标

| 指标 | 目标值 |
|------|--------|
| 从上传到报告完成的时间 | < 3 分钟 |
| 报告可用率（无需手动修改即可使用） | > 70% |
| 转写准确率（中文） | > 90% |
| Demo 可用性（面试官能独立走完流程） | 100% |

## 4. 核心功能（MVP）

### 4.1 功能清单

| 优先级 | 功能 | 描述 |
|--------|------|------|
| **P0** | 语音上传 + AI 转写 | 支持 mp3/wav/m4a，调用 Whisper API 转写为中文文本 |
| **P0** | 文本直接输入 | 粘贴访谈记录，跳过转写步骤 |
| **P0** | AI 结构化分析 | 按 PM 分析框架输出 8 个维度的洞察 |
| **P0** | 分析报告展示 | 可视化呈现所有维度的分析结果 |
| **P1** | 报告历史列表 | 查看、搜索、删除历史报告 |
| **P2** | 用户认证 | 邮箱注册登录，数据隔离 |
| **P2** | 报告分享链接 | 生成可分享的报告链接 |

### 4.2 AI 分析框架（8 大输出维度）

1. **用户目标（JTBD）** — 用户想完成什么任务？
2. **核心痛点** — 按频率和情绪强度排序的痛点列表
3. **使用场景** — 触发条件、上下文、用户环境
4. **现有解决方案** — 用户现在怎么解决的？哪里不满意？
5. **情绪拐点** — 对话中情绪明显变化的时刻（积极/消极）
6. **需求优先级矩阵** — 紧急度 × 重要度 × 用户价值 × 商业价值
7. **用户故事卡片** — "As a [用户], I want to [需求], So that [价值]"
8. **待验证假设** — 哪些说法需要进一步验证？

### 4.3 不做什么（MVP 边界）

- ❌ 多人协作 / 团队空间
- ❌ 自定义分析模板
- ❌ 与 Jira/Linear 等工具打通
- ❌ 竞品对比分析
- ❌ 移动端 App
- ❌ 视频访谈分析

## 5. 产品架构

### 5.1 技术架构

```
┌─────────────────────────────────────────────┐
│                  前端 (Next.js)               │
│  Landing / Analyze / Report / Dashboard      │
│  Tailwind CSS + shadcn/ui                    │
├─────────────────────────────────────────────┤
│               API Routes (Next.js)            │
│  /api/transcribe  /api/analyze  /api/reports │
├─────────────────────────────────────────────┤
│                 AI 层 (OpenAI)                │
│  Whisper (转写)  +  GPT-4o (分析)            │
├─────────────────────────────────────────────┤
│              数据层 (Supabase)                │
│  PostgreSQL + Auth + Storage                 │
└─────────────────────────────────────────────┘
```

### 5.2 核心流程

```
用户上传文件/粘贴文本
        ↓
    [音频文件] → Whisper API → 转写文本
        ↓
    转写文本 / 输入文本
        ↓
    PM System Prompt + 文本 → GPT-4o (JSON Mode)
        ↓
    结构化分析 JSON
        ↓
    存储到 Supabase
        ↓
    渲染报告页面（8 个可视化模块）
```

## 6. 竞品分析

| 竞品 | 定位 | 优势 | 劣势 | InsightFlow 差异化 |
|------|------|------|------|-------------------|
| **飞书妙记** | 会议转写 | 飞书生态、中文好、免费 | 只转写不分析 | 从文字到洞察的跨越 |
| **通义听悟** | 语音转写 | 阿里生态、免费额度大 | 只转写不分析 | PM 专用的分析框架 |
| **Dovetail** | 用户研究平台 | 功能全面、协作强 | 贵($30+/月)、重、英文优先 | 轻量、中文原生、免费起步 |
| **Condense** | 用户研究分析 | 学术严谨 | 学习成本高、不适合 PM | 零学习成本、PM 语言 |
| **ChatGPT** | 通用 AI | 灵活 | 不结构化、无项目管理 | 固化 PM 最佳实践到工作流 |

## 8. 附录

### 8.1 关键设计原则

- **Mobile First**: 响应式设计，手机也能操作
- **Progressive Disclosure**: 先展示概览，再展开细节
- **Human-in-the-loop**: AI 产出可编辑、可修正
- **Privacy First**: 用户数据加密存储，不上传用于模型训练

### 8.2 Glossary

| 术语 | 定义 |
|------|------|
| JTBD | Jobs to Be Done — 用户想要完成的任务 |
| 情绪拐点 | 对话中用户情绪发生明显变化的时刻 |
| PM 分析框架 | InsightFlow 定义的 8 维度结构化分析模板 |
